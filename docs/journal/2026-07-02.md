## Bug fixes (all live-verified)

### 1. CrewAI Flow self-triggering listeners + dead retry caps (`dabef2b`)

Root cause read from installed `crewai/flow/flow.py`, not docs: method completion emits
the method's own name (`_execute_listeners(trigger_method=<name>)`), `_find_triggered_methods`
has no self-exclusion, and `_execute_single_listener` *discards* completed methods "for
cyclic flows". Ten `@listen("X") def X` methods in `main_flow.py` were each an unbounded
self-loop; `finalize_project` even carried a hand-written idempotency guard against the
symptom, cause unknown at the time. Separately: plain `@listen` return values route
nowhere (only `@router` returns do) — `retry_development`'s
`return "escalate_to_human"` was dead code, which is why every retry cap "existed" but
never fired across three handoffs (06-25 "8× duplicate emissions", 06-26 "top priority
infinite loop", 07-01 §9).

Fix: `on_<trigger>` rename convention (trigger strings unchanged), real
`@router(on_retry_development)` carrying the cap, abort-precedence fix in escalation
(a default/timeout "Abort" was silently ignored when metadata pre-seeded
`resume_to="retry_development"`), and a **meta-test** that introspects the Flow class
and fails if any method ever listens to its own name again
(`tests/unit/flows/test_flow_wiring.py`).

**Live proof:** 0 runaway retries across all three comparison runs (previously 93k).

### 2-3. Scope-guardrail false positives on QA output (`1b7bd6d`, `2aa9870`)

Run 1: QA's correct output failed `scope_control_guardrail` at 4-15% relevance vs a 25%
floor — the flagged "scope creep" words were literally *test, coverage, suite,
validation*. Fix: strip fenced code blocks before lexical scoring (code is validated by
the security/quality gates that run next; scope judges prose), pass code-only output.
Run 2: dev-phase false positives gone, but QA prose still scored 18% — floor recalibrated
0.25→0.15 from the observed distributions (correct ≈18%, off-topic <5%).

### 4. Role-adherence false positive: `conftest.py` (`ad6c62e`)

Run 2 discovery: the QA role rule allowed only `test_*.py`; QA wrote `tests/conftest.py`
(standard pytest, verified on disk) and burned 3 retries on "writing production source".
Allowlist `conftest.`/`fixtures` in the pattern.

### 5. Per-run spend contexts + real dashboard numbers (`0fa6e1f`)

The spend guard was a process-global singleton — concurrent Compare runs reset each
other's budgets and combined spend counted against whichever was set last. Now a
`contextvars`-scoped state per execution context (LangGraph: producer thread; CrewAI: its
subprocess) with a `run_id`-keyed registry; `/api/runs/{id}` returns real
`$`/tokens/calls plus artifact-derived metrics (workspace file counts, test files,
normalized smoke verdict); monitor `cost_usd` falls back to the spend registry (LangGraph
runs stop reporting null); CrewAI reports spend through its subprocess result payload.
Concurrency test proves two threads enforce their own ceilings.

Full writeup in [COMPARISON_RESULTS.md](../COMPARISON_RESULTS.md) ("Same-model matrix").
`full-claude` profile (all 9 roles → `anthropic/claude-sonnet-4` via
`TeamProfile.model_overrides`; override chain verified end-to-end) × LangGraph ×

**Headline: the confound is mechanically confirmed. Claude wrote real test suites in
4/4 runs; deepseek wrote zero in 3/3.** The §8 "QA degeneration" class is a model
property, not a LangGraph property. Every remaining blocker was ops/harness:

1. **Provider-routing dialect breakage** (new taxonomy entry): OpenRouter served
   claude-sonnet-4 from Google Vertex, whose Anthropic-translation layer rejects
   tool-call ids → 133 retry-400s per run. A hard pin to provider "Anthropic" 404s —
   this key's endpoint pool is Vertex+Bedrock only (endpoint pools are
   account-specific). Fix: `extra_body={"provider": {"ignore": ["Google"]}}` in
   `langgraph_chat`, validated live with a Bedrock tool round-trip and then run 3's
   0 provider errors (from 133). Two commits (`27ce6bd` wrong pin → `36414b0` corrected
   steering — kept both for honest history).
2. **Spend budgets are model-dependent**: the $5 deepseek-calibrated default killed
   runs 1/2/4 ($5.15, $10.05, $8.03). The run-id-scoped spend guard (yesterday's fix)
   fired correctly and cleanly every time — the guard is the success story, the ceiling
   is config (`AI_TEAM_RUN_BUDGET_USD`).
3. **Quality-gate dependency gap** (new harness finding): the gate runs pytest in the
   *harness* venv and never installs the generated `requirements.txt`. Run 3's tests
   died on `ModuleNotFoundError: flask_sqlalchemy` ×4 modules → retries exhausted →
   `awaiting_human`. deepseek runs only ever dodged this by choosing plain `flask`
   (already installed). Fix queued (below).

Net, in the taxonomy's terms: with the model held constant, failures move up the stack —
provider routing, budget calibration, gate-environment fidelity. "Harness > model >
framework", now with controlled evidence.
