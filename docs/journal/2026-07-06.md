## TL;DR

1. **GUID dirs were not from the web UI.** Compare and `/ws/run` already allocate
   human-readable ids (`2026-07-06_HHMMSS_slug_NN` via `resolve_run_id()`). The litter
   was **LangGraph graph tests** calling `graph.invoke()` with `project_id=str(uuid4())`
   while `thread_id` was a human string — the terminal `manager_report` node wrote
   `ResultsBundle(uuid)` to `./workspace` with no tmp isolation.
2. **Fix: one id everywhere.** `run_id ≡ thread_id ≡ project_id ≡ workspace/<id>/`.
   Enforced at intake, scoped via `RunSession`, post-run moved out of the graph into
   the backend.
3. **136 LangGraph-related tests pass**; re-running graph suites adds **zero** new GUID
   dirs under `workspace/`.

## The problem (observed)

After clearing team output data, `workspace/` repopulated with two naming schemes on
the same day:

| Pattern | Example | Source |
|---------|---------|--------|
| Human-readable | `2026-07-06_195451_disconnect-test_01` | Web UI, WebSocket tests, Compare |
| Full UUID | `01a17a96-197b-4be4-a160-9c7a500de2fe` | LangGraph graph unit/integration tests |

**Repro (pre-fix):**

```bash
uv run pytest tests/unit/backends/langgraph_backend/test_main_graph.py::test_main_graph_invoke_reaches_complete -q
ls workspace/   # → one new UUID dir
```

Ten tests across three files created eight GUID dirs. WebSocket tests created only
dated slug dirs.

**Mechanism:** `build_main_graph()` routed `complete` / `error` → `manager_report` →
`END`. `_node_manager_report` called `ResultsBundle(state["project_id"]).init_dirs()`
using whatever `project_id` was in state — **not** `configurable.thread_id` — and
ignored `RunSession` / `scoped_workspace_dir` when tests bypassed `LangGraphBackend`.

Production `LangGraphBackend` was already mostly correct (`project_id = thread_id`,
`scoped_workspace_dir`). The gap was **direct graph invoke** (tests) and **split
identity** (two fields that could disagree).

## Run identity contract (now enforced)

```
run_id  ≡  configurable.thread_id  ≡  state.project_id
      ≡  ResultsBundle.project_id
      ≡  workspace/<run_id>/
      ≡  output/runs/<run_id>/
```

**Rules:**

- Nothing in the graph mints a `project_id`. Intake binds it from `thread_id`.
- Filesystem side effects go through `RunSession` (contextvar + `scoped_workspace_dir`).
- Post-run manager report runs in the **backend** after terminal phases, not as a graph
  node (pure state machine at `complete` / `error` → `END`).

## What shipped

### 1. `RunSession` (`run_session.py`)

Context-managed session before `invoke` / `stream` / `resume`:

- Sets contextvar consumed by `require_run_id()` and `write_langgraph_manager_report()`
- Wraps `scoped_workspace_dir(workspace/<run_id>)`
- Honors `AI_TEAM_SKIP_POST_RUN` (and per-session `skip_post_run=True` for tests)

### 2. Intake binding (`main_graph.py`)

`_node_intake(state, config)` calls `require_run_id(config, state)` and writes
`project_id` on the first superstep. Mismatched seed state (UUID in `project_id`,
human `thread_id`) is corrected; mismatch is logged as `project_id_thread_mismatch`.

### 3. Post-run lifted out of graph (`post_run.py` + `backend.py`)

- Removed `manager_report` node and edges.
- `write_langgraph_manager_report()` called from `LangGraphBackend` after
  `write_state` / scorecard on `run()`, `resume()`, and `iter_stream_events()`.
- Respects `AI_TEAM_SKIP_POST_RUN` and `RunSession.skip_post_run`.

### 4. Backend entry points (`backend.py`)

`run()`, `resume()`, `iter_stream_events()` all open `_run_session(thread_id, kwargs)`
instead of ad-hoc `scoped_workspace_dir` calls.

### 5. Test harness (`tests/unit/backends/langgraph_backend/`)

| File | Role |
|------|------|
| `harness.py` | `base_graph_state()`, `graph_invoke()` — aligned ids, optional tmp workspace |
| `conftest.py` | Autouse: `AI_TEAM_SKIP_POST_RUN=1`, isolated `PROJECT_OUTPUT_DIR` |
| `test_run_session.py` | Unit tests for identity helpers |

Updated offenders:

- `test_main_graph.py`
- `test_main_graph_full_mocked_e2e.py`
- `integration/test_langgraph_backend_integration.py` (+ test that graph tests do not
  touch repo `workspace/`)

Also: `reset_subgraph_cache()` tolerates monkeypatched cache functions (teardown no
longer crashes when `_cached_planning` is replaced by a lambda).

### 6. CrewAI parity (`flows/state.py`, `main_flow.py`)

- `ProjectState.project_id` default changed from `uuid4()` to `""`.
- `kickoff()` assigns via `resolve_run_id()` when still empty (defense in depth; same
  contract as LangGraph backend).

## Identity flow by caller

| Caller | How run id is chosen |
|--------|----------------------|
| Web `/ws/run` | `resolve_run_id(description, profile)` → `thread_id` |
| CLI `ai-team --backend langgraph` | Same via backend kwargs |
| Compare (3 columns) | One `resolve_run_id` per backend launch |
| HITL resume | Reuse stored `thread_id` — no re-allocation |
| Unit test | Explicit `thread_id` via `graph_invoke()` |
| Evals | `AI_TEAM_SKIP_POST_RUN=1` (unchanged) |

`comparison_id` (frontend `crypto.randomUUID()`) is **SQLite grouping only** — never a
workspace folder name.

## Design note for future backends

When adding a fourth backend, copy this pattern:

1. **One run id** at entry (`resolve_run_id` or explicit override).
2. **`RunSession`-style scoping** — never bare `os.environ["PROJECT_WORKSPACE_DIR"]`
   without a context manager.
3. **Graph / flow nodes** mutate state only; persistence and post-run reports belong in
   the backend wrapper after invoke returns.
4. **Tests** use a harness with tmp workspace + `skip_post_run`, never `uuid4()` for
   `project_id` when `thread_id` is human-readable.
