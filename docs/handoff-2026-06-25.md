# Eval Framework Handoff — 2026-06-25

## What we built

Cross-backend comparison eval that runs crewai / langgraph / claude-agent-sdk in **parallel subprocesses** against a smoke-test scenario, scores them with an LLM judge (Claude Haiku), and prints a comparison table.

```
uv run python -m evals.run_evals --compare --scenario smoke-test -v
# logs per backend: /tmp/eval_{crewai,langgraph,claude_agent_sdk}.log
```

## Known remaining issues

### crewai: RecursionError in json.dumps (likely fixed by bdd366d, not yet confirmed)
`flow.state.model_dump(mode="json")` embeds LangChain `AIMessage` objects with circular `__dict__` refs. Fix: `json.dumps(enriched, default=str)` with `recursionlimit=500` so it fails fast into the fallback dict instead of spinning at 96% CPU.

### crewai: process hangs past project_complete
After `project_complete` log line, the self-improvement crew + `write_scorecard` run. These use additional LLM calls (~60s). Not a deadlock — just slow. Total runtime ~430s for smoke-test.

### langgraph: deepseek tool compliance
deepseek-v3 says "I will use file_writer tool" then outputs markdown prose instead of calling the tool. The `_extract_and_write_code_blocks` fallback parses fenced code blocks — this works when deepseek uses ``` fences. When it doesn't, workspace has only `conftest.py` → pytest exits 5 → retry loop hits max_retries=3. This is an LLM compliance issue, not an eval framework bug. Current passing rate is non-deterministic (~50%).

### SDK: cost
Claude Sonnet via Agent SDK costs $0.30–$0.40 for smoke-test. Budget raised to $0.50 in scenario JSON. Still a real signal: SDK is 30-40x more expensive than crewai/langgraph for this trivial task due to full agent scaffolding.

### SDK: test_pytest_passes_in_workspace
SDK puts files at `ws/src/calc.py` + `ws/tests/test_calc.py`. Our `run_pytest_in_workspace` runs pytest at `ws/` root — old pycache causes "import file mismatch". Fix: `--import-mode=importlib` (committed in 15b2099, not yet tested in a full run).

## How to run

```bash
# Full comparison (all 3 backends, parallel)
uv run python -m evals.run_evals --compare --scenario smoke-test -v

# Single backend
uv run python -m evals.run_evals --backend crewai --scenario smoke-test -v

# Skip LLM judge (faster)
uv run python -m evals.run_evals --compare --no-judge

# Monitor individual backend logs
tail -f /tmp/eval_crewai.log
tail -f /tmp/eval_langgraph.log
tail -f /tmp/eval_claude_agent_sdk.log
```

Requires: `.env` with `ANTHROPIC_API_KEY` and `OPENROUTER_API_KEY`. `AI_TEAM_USE_REAL_LLM=1` is set automatically by the runner.
