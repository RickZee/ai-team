## What Was Fixed (commits on main)

### `8a9ee18` — MockValSer TypeError (LangGraph)
deepseek-v3 via OpenRouter returns `function.arguments` as a dict; openai SDK `defer_build=True` stores it; `model_dump()` triggers `TypeError: 'MockValSer' object cannot be converted to 'SchemaSerializer'`.

**Fix:** httpx response event hook `_fix_chat_completion_response` intercepts the raw HTTP body, converts dict→JSON string before the SDK parses it. Lives in `src/ai_team/backends/langgraph_backend/graphs/langgraph_chat.py`.

### `f67f826` — `_run_success()` for claude-agent-sdk
SDK returns `state=None`; check now falls back to `result.get("success") is True`.

### `ebb7d97` — All-backends smoke test passing

**1. QA role_adherence guardrail too strict**
Pattern `def\s+(?!test_)\w+(` fired on production code QA *quotes* in its analysis (not code QA wrote). Replaced with a narrower pattern targeting `file_writer('non_test.py', ...)` calls only.
- File: `src/ai_team/guardrails/behavioral.py`

**2. Developer model doesn't call `file_writer` tool**
deepseek-v3 describes what it will do instead of actually calling the tool. Added `_extract_and_write_code_blocks()` fallback in `development_subgraph_node`: when workspace is empty post-development, parse `filename.py` + fenced code block pairs from AI message text and write them to disk.
- File: `src/ai_team/backends/langgraph_backend/graphs/subgraph_runners.py`

**3. Quality gate failures**
- `conftest.py` written to workspace root so `from calc import add` resolves when test files are in `tests/` subdirectory
- ruff auto-fix (imports, trailing newlines) runs before scoring
- `pytest --rootdir=<workspace>` prevents picking up parent `pyproject.toml`

**4. QA scope guardrail threshold (min_relevance)**
Code-heavy roles (qa, backend, frontend, fullstack) use 0.25 vs default 0.5. QA test output vocabulary naturally diverges from project description prose.
- File: `src/ai_team/backends/langgraph_backend/graphs/langgraph_guardrail_nodes.py`

**5. Other improvements**
- `phase_history` now written from each subgraph node return
- Testing subgraph context includes project description + workspace path
- `agents.yaml`: stronger `file_writer` instructions in fullstack_developer and qa_engineer backstory

## Known Remaining Issues

### Developer model skips tool calls
deepseek-v3 narrates tool use instead of calling it. The `_extract_and_write_code_blocks` fallback compensates, but it's fragile (regex-based filename detection). Better fix: enforce `tool_choice` at the LangGraph ReAct level, or switch developer to a model with more reliable tool use (e.g., deepseek-r1, claude-sonnet).

### ruff `--preview` flag
`W391` (trailing blank line at end of file) requires `--preview`. Currently enabled in quality gate. If ruff version doesn't support it, suppress the warning or drop W391 from the fix list.

### `__pycache__` in `generated_files`
After running tests, conftest pycache appears in `generated_files` snapshot. Add `.pyc` / `__pycache__` exclusion to `_snapshot_workspace_files()`.
