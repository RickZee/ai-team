### Bugs Fixed (committed: `8a9ee18`, `f67f826`)

#### 1. MockValSer TypeError — LangGraph full mode (FIXED)
**Root cause:** deepseek-v3 via OpenRouter returns `function.arguments` as a parsed dict instead of the JSON string the OpenAI SDK requires. The SDK's `Function` model uses `defer_build=True`, so it stores the dict without validation. When LangGraph's checkpointer later calls `model_dump()` on the `ChatCompletion`, pydantic_core finds `MockValSer` (placeholder serializer) and throws `TypeError: 'MockValSer' object cannot be converted to 'SchemaSerializer'`.

**Fix:** `src/ai_team/backends/langgraph_backend/graphs/langgraph_chat.py`  
Inject an `httpx` response event hook (`_fix_chat_completion_response`) that rewrites `function.arguments` dict→JSON string in the raw HTTP body before the SDK parses the response. This intercepts at the byte level, before pydantic constructs anything.

```python
def _fix_chat_completion_response(response: httpx.Response) -> None:
    if "chat/completions" not in str(response.request.url):
        return
    response.read()
    data = json.loads(response.text)
    # convert any dict arguments → JSON string
    ...
    response._content = json.dumps(data).encode()
```

The hook is installed via `httpx.Client(event_hooks={"response": [_fix_chat_completion_response]})` in `create_chat_model_for_role()`.

**Key learnings:**
- Patching `_create_chat_result` (tried first) is too late — pydantic already stored the bad value
- Custom `httpx.BaseTransport` wrappers break streaming because `response.read()` returns gzip-compressed bytes; the new `httpx.Response` re-applies decompression and fails
- Event hooks run after the response is received, call `response.read()` (decompresses), and can rewrite `response._content` without breaking the SDK

## Remaining Issues (LangGraph)

### Issue 1: QA behavioral guardrail too strict
**Symptom:** `testing` phase errors with `Output deviates from task scope (relevance 35% below 50%)` after 3 retries → `guardrail_terminal` → phase error.

**Location:** `src/ai_team/guardrails/behavioral.py:scope_control_guardrail()`

```python
overlap = len(req_words & out_words) / len(req_words)
# req_words = 4+ char words from project description
# out_words = 4+ char words from agent output
if overlap < min_relevance:  # 0.5 threshold
    return GuardrailResult(status="fail", ...)
```

**Problem:** QA agent writes `pytest` test code. Test code uses words like `assert`, `import`, `pytest`, `test_add` — but the project description uses words like `function`, `returns`, `include`, `calls`, `result`. Overlap is ~35% even when the QA output is correct.

**Fix options:**
1. Lower `min_relevance` from 0.5 → 0.3 for QA/testing subgraph specifically
2. Pass `min_relevance` as param to `make_behavioral_guardrail_node()` and set lower for qa_engineer
3. Exclude QA roles from `scope_control_guardrail` entirely (only run role_adherence + reasoning)
4. Use bigrams or semantic similarity instead of raw keyword overlap

**Recommended:** Option 2 — parameterise threshold per subgraph.

### Issue 2: Development agent not calling `file_writer`
**Symptom:** `generated_files: []` in final state even when development phase completes without error.

**Evidence:** Planning and development subgraphs compile and run, but no files are written to workspace. QA agent then has nothing to test, produces generic output that fails scope check.

**Root cause candidates:**
- deepseek-v3 outputs code as markdown text rather than calling `file_writer` tool
- `file_writer` tool prompt instructions not effective enough
- Tool binding issue (tools not passed to the react agent correctly)

**Check:** `src/ai_team/backends/langgraph_backend/agents/tools.py` — verify `file_writer` is in `get_langchain_tools_for_role("fullstack_developer")`

**Fix:** 
- Verify tool binding: add debug log showing which tools are bound
- Strengthen system prompt in `agents/prompts.py` / `config/agents.yaml` to require `file_writer` calls
- Optionally: add a post-development hook that validates `generated_files` is non-empty, retries if empty

### Issue 3: `phase_history` always empty
Planning/development/testing phases complete but `phase_history: []`. Phase transitions may not be updating state correctly. Check `src/ai_team/backends/langgraph_backend/graphs/routing.py` and main graph node functions.
